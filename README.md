# Mini Smart Warehouse

Мини-приложение для управления складом с умным поиском товаров.
Проект демонстрирует использование нескольких баз данных,
каждая из которых решает свою задачу.

## Цель проекта
Реализовать простой складской сервис, который:
- хранит товары и их остатки,
- поддерживает обычный и семантический поиск,
- использует разные типы БД в рамках одного приложения.

---

## Используемые технологии
- FastAPI
- PostgreSQL — хранение структурированных данных о товарах
- MongoDB — хранение описаний и характеристик товаров
- Redis — кеширование результатов запросов
- Qdrant — семантический поиск и поиск похожих товаров
- RabbitMQ — асинхронная обработка задач
- Neo4j — граф связей товар–категория

---

## Функциональность
- Добавление товара на склад
- Обновление количества товара
- Получение списка товаров
- Поиск товаров по названию и описанию
- Семантический поиск похожих товаров
- Асинхронная генерация эмбеддингов

---

## Использование баз данных

### PostgreSQL
Хранит основные данные о товарах:
- id
- название
- категория
- количество на складе

### MongoDB
Хранит расширенную информацию о товаре:
- описание
- произвольные характеристики

### Redis
Используется для:
- кеширования списка товаров
- кеширования результатов поиска

### Qdrant
Используется для:
- хранения векторных представлений описаний товаров
- поиска похожих товаров

### RabbitMQ
Используется для:
- фоновой обработки новых товаров
- асинхронной генерации эмбеддингов

### Neo4j
Используется для:
- хранения связей между товарами и категориями

---

## Архитектура
FastAPI сервис взаимодействует с несколькими БД,
каждая из которых используется для своего типа данных и нагрузки.
